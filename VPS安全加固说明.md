# VPS安全加固脚本使用说明

## 概述

本脚本是一个专门为Debian/Ubuntu VPS服务器设计的安全加固工具，提供交互式界面，帮助用户快速完成服务器的安全配置。

## 功能特性

### 🔧 主要功能
1. **系统信息显示** - 查看服务器基本信息
2. **系统更新** - 更新系统软件包
3. **用户管理** - 创建非root用户并配置sudo权限
4. **SSH安全配置** - 修改SSH端口、配置root登录策略、高级加密算法
5. **防火墙配置** - 使用ufw配置基础防火墙规则
6. **fail2ban安装** - 防止暴力破解攻击
7. **网络安全配置** - 优化网络安全参数、禁用不必要服务
8. **系统监控配置** - 设置日志轮转和定时检查
9. **精简系统组件** - 移除不必要的软件包和服务
10. **安全扫描与检查** - 端口扫描、用户检查、系统完整性检查
11. **备份与恢复配置** - 配置文件备份和自动备份设置
12. **一键全部执行** - 自动执行所有安全加固措施

### 🛡️ 安全特性
- 操作前确认机制，避免误操作
- 自动备份配置文件
- 详细的操作日志记录
- 兼容性检查
- 分步骤执行，可选择性配置

## 使用方法

### 1. 上传脚本到服务器
```bash
# 使用scp上传（在本地执行）
scp security-hardening.sh root@your-server-ip:/root/

# 或者在服务器上直接创建文件并复制内容
nano security-hardening.sh
```

### 2. 设置执行权限
```bash
chmod +x security-hardening.sh
```

### 3. 运行脚本
```bash
./security-hardening.sh
```

## 详细功能说明

### 1. 系统信息显示
- 显示操作系统版本
- 内核版本和CPU架构
- 内存和磁盘使用情况
- 当前用户和SSH连接信息

### 2. 系统更新
- 更新软件包列表
- 升级所有已安装的软件包
- 清理不需要的软件包
- 清理软件包缓存

### 3. 创建非root用户
- 交互式创建新用户
- 自动添加到sudo组
- 创建SSH目录并设置权限
- 提醒配置SSH密钥

### 4. SSH安全配置
- **修改SSH端口**: 避免默认22端口被扫描
- **Root登录策略**:
  - 完全禁用root登录 (PermitRootLogin no)
  - 仅允许密钥认证 (PermitRootLogin prohibit-password) - 推荐
- **禁用密码认证**: 强制使用SSH密钥
- **高级安全配置**:
  - 限制协议版本为2
  - 设置最大认证尝试次数
  - 配置客户端存活检测
  - 限制登录宽限时间和并发连接数
  - 强化加密算法 (curve25519, AES-GCM, ChaCha20)
  - 禁用不安全的认证方式
  - 限制用户登录权限

### 5. 防火墙配置
- 安装并配置ufw防火墙
- 设置默认拒绝入站策略
- 自动允许SSH端口
- 可选择开放HTTP/HTTPS端口
- 支持自定义端口配置
- 显示防火墙规则状态

### 6. fail2ban安装配置
- 安装fail2ban服务
- 配置SSH保护规则
- 设置封禁时间和重试次数
- 支持nginx保护（可选）
- 自动启动服务

### 7. 网络安全配置
- **内核参数优化**:
  - 禁用IP转发
  - 禁用源路由
  - 禁用重定向
  - 启用反向路径过滤
  - 忽略ICMP ping请求
  - 启用SYN cookies
- **服务检查**: 检查并禁用不必要的网络服务

### 8. 系统监控配置
- 配置日志轮转
- 安装监控工具（htop, iotop, nethogs）
- 设置每日安全检查脚本
- 监控失败登录尝试
- 记录sudo使用情况

### 9. 精简系统组件
- **移除不必要软件包**:
  - 网络服务 (telnet, rsh, finger, talk等)
  - 过时的网络工具
  - 不需要的系统服务
- **禁用不必要服务**:
  - avahi-daemon (网络发现)
  - cups (打印服务)
  - bluetooth (蓝牙)
  - snapd (snap包管理)
- **系统清理**:
  - 清理包缓存和临时文件
  - 清理旧日志文件
  - 自动移除孤立软件包

### 10. 安全扫描与检查
- **端口扫描**: 检查开放端口和监听服务
- **用户账户检查**:
  - 检查空密码账户
  - 检查UID为0的账户
  - 列出可登录用户
- **文件权限检查**: 验证关键系统文件权限
- **登录记录分析**: 检查最近登录和失败尝试
- **系统完整性扫描**: 使用rkhunter检测rootkit和异常

### 11. 备份与恢复配置
- **完整配置备份**:
  - SSH配置文件
  - 防火墙规则
  - 用户账户信息
  - 系统配置文件
  - 已安装软件包列表
- **自动备份设置**: 每周自动备份重要配置
- **备份管理**: 自动清理过期备份文件

## 重要注意事项

### ⚠️ 使用前必读

1. **备份重要数据**: 脚本会自动备份配置文件，但建议提前备份重要数据
2. **SSH连接**: 修改SSH配置前，确保了解新的连接方式
3. **防火墙规则**: 配置防火墙时注意不要阻断当前SSH连接
4. **密钥认证**: 禁用密码认证前，确保已正确配置SSH密钥

### 🔑 SSH密钥配置示例

在本地生成SSH密钥对：
```bash
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

将公钥复制到服务器：
```bash
ssh-copy-id -p 端口号 用户名@服务器IP
```

### 📁 文件位置

- **脚本位置**: `/root/security-hardening.sh`
- **日志文件**: `/var/log/security-hardening.log`
- **备份目录**: `/root/security-backup-YYYYMMDD-HHMMSS/`
- **每日检查日志**: `/var/log/daily-security-check.log`

## 故障排除

### 常见问题

1. **SSH连接断开**
   - 检查新的SSH端口是否正确
   - 确认防火墙规则是否允许新端口
   - 使用控制台或VNC连接服务器

2. **权限问题**
   - 确保以root用户运行脚本
   - 检查文件权限设置

3. **服务启动失败**
   - 查看系统日志: `journalctl -xe`
   - 检查配置文件语法

### 恢复配置

如果需要恢复原始配置：
```bash
# 查看备份目录
ls /root/security-backup-*/

# 恢复配置文件
cp /root/security-backup-*/sshd_config /etc/ssh/
systemctl restart sshd
```
