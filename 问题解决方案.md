# VLESS-TCP-HTTP-REALITY 问题解决方案

## 🔧 已修复的核心问题

### 1. 协议兼容性问题
- **问题**: 新版Xray移除了传统HTTP传输，导致配置失败
- **解决**: 改用TCP+HTTP伪装，兼容性更好
- **修改**: `network: "h2"` → `network: "tcp"` + `tcpSettings`

### 2. 端口占用问题  
- **问题**: 端口443被占用导致服务启动失败
- **解决**: 添加端口检查和自动释放功能
- **功能**: 自动检测、强制释放、重新分配端口

## 📋 你的三个问题解答

### 问题1: 自定义域名时是否需要改配置文件中的域名？

**答案: 是的，需要同步更改多个位置**

现在脚本已自动处理，当你选择自定义域名时，会自动更新：
- `realitySettings.serverNames[]` - REALITY SNI域名
- `realitySettings.dest` - 目标服务器地址  
- `tcpSettings.header.request.headers.Host[]` - HTTP伪装Host头

**使用方法:**
```bash
# 运行脚本后选择选项3
./vps.sh
# 选择: 3. 更改域名配置
```

### 问题2: 重新生成订阅时是否要更换旧订阅并kill旧端口？

**答案: 是的，已自动处理**

新增功能会自动：
1. **备份旧配置** - 自动备份为 `.backup-时间戳` 文件
2. **停止旧服务** - `systemctl stop xray`
3. **释放端口** - 强制终止占用进程
4. **更新配置** - 生成新的客户端配置
5. **重启服务** - 使用新配置启动

**使用方法:**
```bash
# 重新生成客户端配置
./vps.sh
# 选择现有配置管理选项
```

### 问题3: 可以提供设置更改旧的UUID？

**答案: 已提供完整的UUID管理功能**

新增的UUID管理功能：

#### 3.1 添加新用户
- 生成新UUID
- 指定用户邮箱标识
- 自动更新配置并重启服务

#### 3.2 删除用户  
- 选择要删除的用户编号
- 保护机制：至少保留一个用户
- 自动更新配置

#### 3.3 重新生成所有UUID
- 指定新的用户数量(1-10)
- 批量生成新UUID
- 完全替换旧配置

**使用方法:**
```bash
./vps.sh
# 选择: 2. 更新用户UUID
# 然后选择具体操作(添加/删除/重新生成)
```
