# VPS快速安全加固指南

## 🚀 快速开始

### 1. 上传并运行脚本
```bash
# 上传脚本到服务器
scp security-hardening.sh root@your-server-ip:/root/

# 登录服务器
ssh root@your-server-ip

# 设置执行权限并运行
chmod +x security-hardening.sh
./security-hardening.sh
```

### 2. 推荐的安全加固顺序

#### 🔥 紧急优先级 (立即执行)
1. **系统更新** (选项2)
   - 修复已知安全漏洞
   - 更新系统组件

2. **创建非root用户** (选项3)
   - 创建管理员用户
   - 配置sudo权限
   - 设置SSH密钥

3. **SSH安全配置** (选项4)
   - 修改SSH端口 (推荐: 2222, 2022, 或其他非标准端口)
   - 设置root登录策略 (推荐: prohibit-password)
   - 启用高级安全配置

#### 🛡️ 高优先级 (第一天完成)
4. **防火墙配置** (选项5)
   - 启用ufw防火墙
   - 配置基础规则
   - 开放必要端口

5. **安装fail2ban** (选项6)
   - 防止暴力破解
   - 自动封禁恶意IP

#### 🔧 中优先级 (一周内完成)
6. **网络安全配置** (选项7)
   - 优化内核参数
   - 禁用不必要服务

7. **精简系统组件** (选项9)
   - 移除不必要软件
   - 禁用无用服务

#### 📊 低优先级 (持续维护)
8. **系统监控配置** (选项8)
   - 设置日志监控
   - 配置定时检查

9. **安全扫描检查** (选项10)
   - 定期安全扫描
   - 系统完整性检查

10. **备份配置** (选项11)
    - 配置自动备份
    - 测试恢复流程

## 📋 关键配置检查清单

### SSH安全检查
- [ ] SSH端口已修改 (非22端口)
- [ ] Root登录策略已配置 (prohibit-password或no)
- [ ] 密码认证已禁用 (仅密钥认证)
- [ ] SSH密钥已正确配置
- [ ] 允许登录的用户已限制

### 防火墙检查
- [ ] ufw防火墙已启用
- [ ] 默认策略: 拒绝入站，允许出站
- [ ] SSH端口已开放
- [ ] 仅开放必要端口
- [ ] fail2ban已安装并运行

### 用户管理检查
- [ ] 已创建非root管理员用户
- [ ] 管理员用户已加入sudo组
- [ ] 无空密码账户
- [ ] 无不必要的系统用户

### 系统安全检查
- [ ] 系统已更新到最新版本
- [ ] 不必要的软件包已移除
- [ ] 不必要的服务已禁用
- [ ] 网络安全参数已优化
- [ ] 日志监控已配置

## ⚡ 一键执行模式

如果您是新手或希望快速完成所有配置，可以使用一键执行模式：

```bash
./security-hardening.sh
# 选择选项 12 (一键全部执行)
```

**注意**: 一键执行会依次执行所有安全加固措施，请确保：
- 您已准备好SSH密钥
- 了解将要修改的配置
- 有备用的服务器访问方式

## 🔧 常用命令参考

### SSH连接 (端口修改后)
```bash
ssh -p 新端口号 用户名@服务器IP
```

### 防火墙管理
```bash
# 查看防火墙状态
sudo ufw status numbered

# 开放端口
sudo ufw allow 端口号/tcp

# 删除规则
sudo ufw delete 规则编号
```

### fail2ban管理
```bash
# 查看状态
sudo fail2ban-client status

# 查看SSH保护状态
sudo fail2ban-client status sshd

# 解封IP
sudo fail2ban-client set sshd unbanip IP地址
```

### 系统监控
```bash
# 查看系统负载
htop

# 查看网络连接
sudo nethogs

# 查看磁盘IO
sudo iotop

# 查看登录记录
last -n 20

# 查看失败登录
grep "Failed password" /var/log/auth.log
```

## 🚨 紧急情况处理

### SSH连接问题
1. **无法连接**: 检查新端口号和防火墙规则
2. **密钥认证失败**: 确认密钥文件权限 (600)
3. **用户被锁定**: 使用控制台或VNC连接

### 恢复配置
```bash
# 查看备份目录
ls /root/security-backup-*

# 恢复SSH配置
sudo cp /root/security-backup-*/sshd_config /etc/ssh/
sudo systemctl restart sshd

# 恢复防火墙
sudo ufw --force reset
sudo ufw disable
```

### 紧急访问
如果SSH无法连接：
1. 使用云服务商的控制台/VNC
2. 检查防火墙规则
3. 恢复SSH配置
4. 重启SSH服务

## 📞 技术支持

遇到问题时的排查步骤：
1. 查看脚本日志: `/var/log/security-hardening.log`
2. 查看系统日志: `journalctl -xe`
3. 检查服务状态: `systemctl status 服务名`
4. 查看网络连接: `ss -tlnp`
