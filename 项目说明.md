# VPS 安全加固和代理搭建项目

## 项目概述

本项目提供一个统一的 VPS 管理脚本，集成了安全加固和代理服务搭建的完整功能。

## 核心文件

### 主要脚本
- `security-hardening.sh` - **核心脚本**，包含所有功能

### 主要文档
- `fail2ban完整指南.md` - **主要文档**，完整使用指南
- `README.md` - 项目介绍和快速开始

### 专项文档
- `VPS安全加固说明.md` - 安全功能详细说明
- `SSH配置最佳实践.md` - SSH 安全配置
- `vless+reality配置指南.md` - 代理配置指南

## 功能特点

### 🛡️ 安全加固 (选项1-13)
- 系统更新和用户管理
- SSH 安全配置
- 防火墙配置 (ufw)
- fail2ban 安装和管理
- 网络安全配置
- 系统监控和清理

### 🚀 代理服务 (选项14-20)
- 证书管理 (Cloudflare DNS)
- Hysteria2 服务
- X-UI 面板
- Sub-Store 服务
- Nginx 分流配置
- vless+reality 代理

### 🔧 智能特性
- 自动检测系统配置
- 智能错误修复
- 分级配置策略
- 详细状态反馈

## 使用方法

### 基本使用
```bash
chmod +x security-hardening.sh
./security-hardening.sh
```

### 推荐流程
1. **安全加固**: 选择选项21 (一键执行) 或依次执行1-6
2. **证书申请**: 选择选项14，申请 SSL 证书
3. **服务安装**: 根据需要选择15-17安装代理服务
4. **分流配置**: 选择选项18配置 Nginx 分流

## 系统要求

- **操作系统**: Debian/Ubuntu
- **权限**: root 用户
- **网络**: 能够访问外网
- **域名**: 使用代理功能需要域名 (推荐 Cloudflare 托管)

## 安全建议

1. **定期更新**: 保持系统和软件包最新
2. **强密码**: 使用复杂密码和SSH密钥
3. **监控日志**: 定期检查安全日志
4. **备份配置**: 重要配置文件的备份
5. **防火墙**: 只开放必要端口

## 故障排除

### 常见问题
- **fail2ban socket 错误**: 使用选项12→7诊断修复
- **SSH端口检测错误**: 已自动修复
- **配置验证失败**: 自动降级到可用配置
- **证书申请失败**: 检查 Cloudflare 配置

### 获取帮助
1. 查看 `fail2ban完整指南.md` 详细说明
2. 使用脚本内置的诊断功能
3. 检查系统日志: `journalctl -u 服务名`
