# VPS 脚本整合说明

## 整合概述

根据您的要求，已将原本分离的多个脚本整合为单一的 `security-hardening.sh` 脚本，避免在 VPS 上管理多个脚本的复杂性。

## 整合内容

### 原有脚本
- ❌ `build-all.sh` - 代理服务搭建脚本 (已删除)
- ❌ `run.sh` - 统一调用脚本 (已删除)
- ✅ `security-hardening.sh` - 主脚本 (已扩展)

### 新增功能模块

#### 1. 证书管理 (选项14)
- 基于 `build-all.sh` 的 `handle_cert` 函数
- 支持 Cloudflare DNS 自动申请证书
- 证书查看、续期、删除功能
- 自动配置证书路径

#### 2. Hysteria2 服务 (选项15)
- 基于 `build-all.sh` 的 `handle_hysteria` 函数
- 自动安装和配置 Hysteria2
- 支持自定义端口和认证
- 自动配置防火墙规则

#### 3. X-UI 面板 (选项16)
- 基于 `build-all.sh` 的 `handle_xui` 函数
- 安装 3X-UI 管理面板
- 提供配置指导
- 支持备份恢复

#### 4. Sub-Store 服务 (选项17)
- 基于 `build-all.sh` 的 `handle_substore` 函数
- 自动安装 Node.js 环境
- 配置 Sub-Store 服务
- 自动创建系统服务

#### 5. Nginx 分流配置 (选项18)
- 基于 `build-all.sh` 的 `handle_nginx` 函数
- 智能 SNI 分流配置
- 支持多域名代理
- 自动 SSL 配置

## 使用优势

### 2. 功能完整

### 3. 易于使用

## 使用流程

### 基础安全加固
```bash
./security-hardening.sh
# 选择: 21 (一键全部执行)
# 或依次选择: 2 → 3 → 4 → 5 → 6
```

### 代理服务搭建
```bash
./security-hardening.sh

# 步骤1: 证书申请
# 选择 14 → 1，输入域名和 Cloudflare 信息

# 步骤2: 服务安装 (根据需要)
# 选择 15 (Hysteria2) 或 16 (X-UI) 或 17 (Sub-Store)

# 步骤3: 配置分流
# 选择 18，配置 Nginx 分流
```

## 配置要求

### Cloudflare 证书申请
需要准备以下信息：
- 域名 (建议使用二级域名)
- 邮箱地址
- Cloudflare API Token
- Cloudflare Zone ID
- Cloudflare Account ID

### 服务配置
- **Hysteria2**: 域名、端口、用户名、密码
- **X-UI**: 手动配置或备份恢复
- **Sub-Store**: API 密钥
- **Nginx**: 主域名、子域名、端口配置

## 注意事项

### 1. 执行顺序
- 建议先完成安全加固再配置代理服务
- 证书申请应在服务安装之前完成
- Nginx 分流配置应在最后进行

### 2. 端口管理
- 确保所需端口未被占用
- 及时配置防火墙规则
- 记录各服务的端口配置

### 3. 证书路径
- 证书自动保存到 `/home/用户名/` 和 `/root/cert/域名/`
- X-UI 等服务需要手动配置证书路径
- 证书会自动续期

### 4. 服务管理
- 所有服务都配置为系统服务
- 使用 `systemctl` 管理服务状态
- 日志位于 `/var/log/` 目录

## 故障排除

### 常见问题
1. **fail2ban socket 错误**: 使用选项 12 → 7 诊断修复
2. **证书申请失败**: 检查 Cloudflare 配置和网络连接
3. **服务启动失败**: 查看系统日志和端口占用情况
4. **Nginx 配置错误**: 检查域名和端口配置

### 日志查看
```bash
# 查看脚本日志
tail -f /var/log/security-hardening.log

# 查看服务状态
systemctl status 服务名

# 查看服务日志
journalctl -u 服务名 -f
```
